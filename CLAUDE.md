## What This Project Is

A plant disease object detection system for a college final presentation (May 11, 2026). Uses the PlantDoc dataset (2,569 images, 30 classes) with a **two-model pipeline**:
- **YOLOv11** for detection (locating diseased leaves with bounding boxes)
- **Vision Transformer (google/vit-base-patch16-224)** for classification (identifying the disease)
- **FastAPI web dashboard** for live demo during the presentation

## Commands

```bash
# Install dependencies
pip install -r requirements.txt

# Pre-download ViT backbone (do this ONCE before going offline/to lab)
python download_weights.py

# --- Training ---

# 1. Train YOLO (run on college lab GPU)
python train_yolo.py
python train_yolo.py --epochs 100 --imgsz 640 --batch 16

# 2. Extract crops from YOLO labels (one-time, before ViT training)
python train_vit.py --extract-crops

# 3. Train ViT
python train_vit.py
python train_vit.py --epochs 20 --batch 32

# --- Evaluation ---
python evaluate.py

# --- Prediction (CLI) ---
python predict.py path/to/leaf.jpg --show --attention

# --- Web Dashboard (for live presentation demo) ---
python web/app.py
# Then open http://localhost:8000
```

## Training Workflow (in order)

**Option A — College lab GPU:**
1. `python train_yolo.py`
2. `python train_vit.py --extract-crops`
3. `python train_vit.py`
4. `python evaluate.py`
5. `python prepare_demo.py` — pre-caches results for demo gallery
6. `python web/app.py`

**Option B — Google Colab free GPU (can do right now):**
- Open `colab_train.ipynb` in Colab, select T4 GPU, run all cells
- Downloads `trained_weights.zip` at the end
- Extract and copy weights to `runs/` on your local machine
- Then run `prepare_demo.py` and `web/app.py` locally

## Key Design Decisions

- **ViT crops**: The ViT is not trained on full images but on bounding-box crops extracted from the YOLO label annotations. This is `extract_crops()` in `utils/dataset_utils.py`.
- **Lazy model loading**: `web/app.py` loads models on first request, so the server starts instantly even if weights aren't trained yet.
- **Attention heatmap**: `get_attention_map()` in `vit_classifier.py` hooks the last ViT attention layer, averages over heads, takes the `[CLS]→patches` attention row, and overlays it as a heatmap on the input image. This is very visual for a presentation.
- **No paid services**: everything runs locally. The server runs on `localhost:8000` during presentation.

## Configuration

All parameters live in `config/config.yaml`. No hardcoded values in source files. Key fields:
- `yolo.trained_weights` — path to the best YOLO checkpoint (used by evaluate.py and web/app.py)
- `vit.checkpoint` — path to the best ViT checkpoint
- `dataset.crops_dir` — where extracted crops live
- `web.confidence_threshold` — YOLO confidence cutoff for the web demo

## Dataset

PlantDoc, YOLO format. 30 classes including 8 healthy leaf types and 22 disease types across 13 plant species (Apple, Tomato, Potato, Corn, Grape, etc.). Labels: `class_id x_center y_center width height` (normalized [0,1]).
